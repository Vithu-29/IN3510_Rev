1. 4G & LTE Overview

What is 4G?

4G (Fourth Generation) provides true mobile broadband with high-speed IP-based services anywhere, anytime. Defined by WWRF, it delivers:

Key Specifications:
  • Speed While Moving: ~100 Mbps
  • Speed While Stationary: ~1 Gbps
  • Network Type: Fully Packet-Switched (All-IP)
  • Access: Seamless Wi-Fi + Cellular integration

The MAGIC Concept

4G capabilities are described by the acronym MAGIC:

  • Mobile Multimedia
  • Anytime Anywhere
  • Global Mobility Support
  • Integrated Wireless Solutions
  • Customized Personal Services

Features of 4G Technology

  • Very high data rates (100 Mbps – 1 Gbps)
  • High Quality of Service (QoS) and reliability
  • Enhanced security mechanisms
  • Low latency for real-time applications
  • Supports HD video streaming, gaming, video calls
  • Mobile broadband everywhere (Anytime, Anywhere)

LTE (Long Term Evolution)

LTE is an advanced 4G mobile broadband standard. Based on UMTS but redesigned for All-IP performance. It offers higher data rates, low latency, and simplified architecture for faster deployment.

Advantages of LTE for Network Operators

  • High network throughput
  • Low latency → faster response
  • Simplified all-IP architecture
  • Lower operational costs
  • Easy upgrade path from 3G

Advantages for End Users

  • Faster downloads & uploads
  • Better video streaming & gaming
  • Improved app responsiveness
  • Better voice/video call quality (VoLTE)

Major LTE Radio Technologies

  • OFDM for Downlink – better spectral efficiency
  • SC-FDMA for Uplink – reduces handset power consumption
  • MIMO – increases capacity and throughput
  • Improved battery life due to efficient RF usage

Drawbacks of 4G

  • Higher battery consumption compared to 3G
  • Requires complex and expensive hardware
  • Challenging deployment & infrastructure cost
  • Not fully compatible with older devices

Comparison: 3G vs 4G

Parameter 3G 4G
Data Rate 0.5–3 Mbps 2–100+ Mbps (up to 1 Gbps peak)
Bandwidth 5–20 MHz Up to 100 MHz
Frequency Band 1.6–2 GHz 2–8 GHz
Tech UMTS / W-CDMA LTE / OFDM / MIMO

Applications of 4G

  • HD video streaming (YouTube, Netflix)
  • Real-time gaming & cloud gaming
  • High-speed mobile internet access
  • VoLTE (High-quality voice & video calls)
  • IoT and smart home connectivity
  • Video conferencing (Zoom, Teams)
  • Augmented reality applications

2. LTE Network Architecture

LTE uses a flat, simplified, all-IP architecture. Unlike 3G, LTE removes the RNC; its functions are moved into the eNodeB, reducing latency and increasing efficiency. LTE is divided into: E-UTRAN (radio) and EPC (core network).

eNodeB (Evolved NodeB)
  • Handles all radio functions (RRC, scheduling, handover decisions).
  • Performs inter-cell mobility and admission control.
  • No RNC — provides a flat architecture with lower latency.
  • Manages uplink SC-FDMA and downlink OFDM signals.
MME (Mobility Management Entity)

Control Plane Only

  • NAS signaling between UE ↔ Core Network.
  • NAS layer functions: manages session establishment & maintains UE communication continuity.
  • NAS security – integrity protection & encryption of NAS messages.
  • Gateway selection – selects appropriate SGW & PGW for UE data sessions.
  • MME selection during inter-MME handovers.
  • SGSN selection for 2G/3G inter-RAT handovers.
  • Roaming management for visiting subscribers.
  • Authentication of the user.
  • Authorization for services & access.
  • Warning message transfer (ETWS/CMAS), including choosing correct eNodeB.
S-GW (Serving Gateway)

User Plane

  • Local mobility anchor for inter-eNodeB handovers.
  • Mobility anchoring for inter-3GPP mobility (e.g., LTE ↔ UMTS/GSM).
  • Packet routing & forwarding between eNodeB and PDN-GW.
  • Transport-level packet marking (uplink & downlink).
  • Accounting for inter-operator charging based on usage.
P-GW (PDN Gateway)

User Plane

  • Main connection point to external networks (Internet, IMS, corporate PDNs).
  • UE IP address allocation for each data session.
  • Lawful interception support for regulatory needs.
  • DHCPv4 & DHCPv6 (server and client functions).
  • Transport-level packet marking in UL & DL.
  • UL/DL service-level rate enforcement (QoS enforcement).
  • Packet screening / filtering (firewall, deep packet inspection).
  • Mobility anchor for non-3GPP access (Wi-Fi, WiMAX).

LTE Architecture Summary

  • E-UTRAN: UE ↔ eNodeB (radio layer).
  • EPC: MME (control), S-GW (user), P-GW (user).
  • Flat all-IP architecture → lower latency & higher throughput.
  • RNC is removed → simplified design.
  • Supports seamless mobility (intra-LTE & inter-RAT).

3. Sensor Networks

Sensor

  • A sensor produces an output (signal) according to a physical input quantity.
  • Acts as an input device providing data to a main control system.

Sensor Network

  • Large networks of simple sensors.
  • Usually deployed randomly.
  • Highly prone to failures.
  • Communicate using broadcast paradigms.
  • Collect and send information to a base station.
  • Focus on power conservation over QoS.

Wireless sensor networks perform large-scale physical measurements using many sensors that send data wirelessly to a base station.

Types of Sensor Networks

  • Static: Sensors remain in place after deployment.
  • Dynamic: Sensors can move on their own or via an external force.

Sensor Node Components

  • Sensing unit
  • Processing unit
  • Storage
  • Transceiver
  • Power unit
  • Optional: Localization, Mobility, Energy Scavenging

Localization in WSN

Localization determines the geographical location of sensor nodes within a wireless sensor network.

Sensor Characteristics

  • Low power consumption
  • Autonomous operation
  • High volumetric deployment
  • Environment adaptive (e.g., vehicle collision avoidance)
  • Low cost

Transceiver Unit

  • Radio Frequency (RF): Larger, more expensive, may cause interference, supports omni-directional or directional antennas.
  • Optical (Smart Dust): Requires line-of-sight, no interference, extremely small.

What Are Motes?

Motes are the building blocks of Wireless Sensor Networks.

  • Mini low-power computer
  • Connected to one or more sensors (temperature, sound, vibration, pressure, etc.)
  • Communicates using a radio link

4. Wireless Sensor Networks (WSN)

Wireless Sensor Network — “A wireless network consisting of spatially distributed autonomous devices using sensors to cooperatively monitor physical or environmental conditions (temperature, sound, vibration, pressure, motion, pollutants, etc.) at different locations.”

Overview

  • Formed by hundreds or thousands of motes that communicate and forward data hop-by-hop.
  • Designed for low-power, autonomous operation and high-volume deployment.
  • Research focus: energy-aware computing and distributed algorithms.

Key Characteristics

  • Low power consumption (sleep/wake cycles).
  • Prone to failures — nodes must self-organize and recover.
  • Broadcast communication paradigm; data collected to a base station.
  • Prioritize power conservation over strict QoS.

TinyOS (WSN OS)

TinyOS is an open-source, component-based OS for wireless embedded sensor networks (developed at UC Berkeley with Intel Research). It emphasizes low complexity and power conservation, is written in nesC, and is tailored for motes and Tiny hardware.

Note: historically referenced stable version: TinyOS 2.1.2 (Aug 2012).

Typical Targets / Motes

  • mica, mica2, mica2dot, micaz
  • telosb, pc-rene2, micaz, Mica2 (popular mote)
  • Mica2: 8-bit AVR controller, FSK radio, data-logger flash — widely used in early deployments.

Habitat Monitoring — Great Duck Island

Intel Research Lab (Berkeley) collaborated with College of the Atlantic to deploy WSN on Great Duck Island (Maine, 2002). Purpose: monitor microclimates around nesting burrows (Leach's Storm Petrel). Goal: provide a habitat-monitoring kit for researchers.

Project URL historically: greatduckisland.net

FireBug — Wildfire Instrumentation

FireBug is a GPS-enabled wireless thermal sensor network (TinyOS-based) for wildfire monitoring. Motes self-organize to collect real-time data used for predictive analysis of fire behavior. Architecture includes sensor layer, processing, and a command center (UC Berkeley project).

BP Experiment — Preventive Maintenance (North Sea)

Collaboration between Intel & BP: sensor networks deployed on an oil tanker to support preventive maintenance. The system collected data reliably and recovered from errors — recognized among top IT projects (2004).

Quick facts

  • WSNs are built from motes (sensor + processing + transceiver + power) cooperating to monitor environments.
  • Primary challenges: energy management, fault tolerance, localization, and efficient data aggregation.
  • Representative hardware: Mica2, Micaz, TelosB; common OS: TinyOS (nesC).
