1. Introduction to Mobile IP

What is Mobile IP?

  • Mobile IP is an enhancement to the standard Internet Protocol (IP).
  • Allows mobile devices to move across different networks.
  • Ensures the device keeps a permanent and stable IP address.
  • Provides continuous communication even when changing network attachment points.
  • Supports seamless mobility without breaking existing connections.

Why Do We Need Mobile IP?

  • Standard IP requires a device to change its IP when moving to a new subnet.
  • Changing IP breaks active TCP connections.
  • DNS updates are slow and inefficient for mobility.
  • Manual routing table changes do not scale and create security risks.

How Mobile IP Solves the Problem

  • Uses forwarding mechanisms to deliver packets to the Mobile Node (MN).
  • Keeps the home IP address unchanged, ensuring transparency for applications.
  • Enables seamless handover when moving across networks.
  • Maintains active TCP/UDP sessions without interruption.
  • Ensures reliable and continuous connectivity while on the move.

Key Requirements of Mobile IP

Transparency: Mobility must be invisible to applications and upper-layer protocols. The device keeps the same IP address and communication continues seamlessly.

Compatibility: Works with existing IPv4 infrastructure. No changes required in fixed hosts or routers.

Security: Registration and mobility messages must be authenticated to prevent impersonation or misuse.

Efficiency & Scalability: Uses minimal signaling—important for low-bandwidth mobile links. Must scale globally to support millions of mobile nodes.

2. Architecture & Terminology

Mobile Node (MN)

  • A host/device that moves across networks.
  • Keeps the same permanent IP address (Home Address).
  • Can change its point of attachment without breaking connections.

Home Agent (HA)

  • Router in the Home Network of the Mobile Node.
  • Maintains the MN's current location (Care-of Address).
  • Intercepts packets sent to MN and tunnels them to the COA.

Foreign Agent (FA)

  • Router in the Foreign Network where MN currently resides.
  • Receives tunneled packets from the HA.
  • Forwards packets to the MN—often the MN’s default router.

Care-of Address (CoA)

  • Temporary address identifying the MN’s current location.
  • Acts as the tunnel endpoint for Mobile IP packets.
  • Two types of CoA:
    • Foreign Agent CoA: IP address of the FA.
    • Co-located CoA: MN gets a temporary IP (e.g., via DHCP) directly from foreign network.

Correspondent Node (CN)

  • The communication partner of the Mobile Node.
  • Can be a fixed device or another mobile device.

Data Transfer in Mobile IP

Data Transfer 'TO' Mobile Node (Downlink)

  • CN sends packets to MN’s Home Address.
  • HA intercepts the packets (using proxy ARP).
  • HA tunnels the packets to the MN's CoA (e.g., FA).
  • FA decapsulates and delivers packets to the MN.

Data Transfer 'FROM' Mobile Node (Uplink)

  • MN sends packets directly to the CN.
  • FA acts as the default router for outgoing packets.
  • Packets are routed normally through the Internet.
  • No tunneling required for uplink traffic.

3. Operations: Registration & Data Transfer

Agent Advertisement

  • HA and FA periodically broadcast ICMP Agent Advertisement messages.
  • MN determines if it is:
    • in Home Network
    • or Foreign Network
  • MN extracts Care-of Address (CoA).
  • Triggers mobility detection & registration.

Registration Process (MN → FA → HA)

  1. Discovery: MN gets CoA from FA.
  2. Request: MN → HA via FA (limited lifetime).
  3. Authentication: Required for security.
  4. Binding: HA maps MN Home IP → CoA.
  5. Reply: HA sends back acknowledgment.

Purpose: Tell HA about MN’s new location.

Encapsulation (Tunneling)

  • HA wraps original IP packet inside a new IP header.
  • Outer header → destination = CoA.
  • Inner header → destination = MN Home Address.
  • Types:
    • IP-in-IP (mandatory)
    • Minimal encapsulation (optional)

Triangular Routing

CN → HA → MN (long path), MN → CN (short path). Creates delays and extra load.

Reverse Tunneling

  • MN → FA
  • FA tunnels to HA
  • HA → CN

Fixes firewall & TTL issues, but increases delay.

Change of Foreign Agent

  • Packets may be lost during movement.
  • New FA informs old FA to forward pending packets.
  • Old FA clears resources after handoff.

4. Mobile IP vs IPv6

Problems with Mobile IP

  • Security issues
    • Authentication with FA is difficult (belongs to another organization).
    • No standardized key management or key distribution.
    • Patent & export restrictions complicate security deployment.
  • Firewalls
    • Mobile IP often does not work through firewalls.
    • Special setups like reverse tunneling are needed.
  • QoS (Quality of Service)
    • Resource Reservation Protocol (RSVP) requires many new reservations.
    • Tunneling makes it difficult to classify & prioritize packet flows.
  • Overall: Security, firewalls, QoS remain active research challenges.

Mobile IP in IPv6

  • Built-in Security
    • Security integrated into IPv6—not an add-on.
    • Registration messages must be authenticated.
    • Nodes use strong authentication & encryption by default.
  • Easy Address Assignment
    • Care-of Address can be auto-configured.
    • DHCPv6 or Stateless Address Auto-Configuration (SLAAC).
  • No Foreign Agent Needed
    • All routers send Router Advertisements.
    • These replace Agent Advertisements used in IPv4 MIP.
  • Direct Route Optimization
    • MN can inform CN directly about its CoA.
    • Packets do not need to go through HA → no triangular routing.
  • Soft Handover (No Packet Loss)
    • MN sends new CoA to old router.
    • Old router forwards buffered packets to new CoA.
    • Always authenticated.

Why IPv6 Simplifies Mobile IP

  • CoA can be generated using IPv6 address auto-configuration.
  • No need for Foreign Agents—mobile nodes work independently.
  • MIPv6 avoids tunneling as much as possible.
  • Routing optimized by letting MN notify CN directly.
  • Improved security, authentication, and mobility support.
  • Overall: IPv6 → simpler, faster, and more secure mobility.

5. Wireless LANs (WLAN)

What is a Wireless LAN?

  • Flexible data communication system using radio waves.
  • Extends or replaces wired LAN.
  • Supports mobility + connectivity for users.

Development Focus

  • Pros/cons and practical uses.
  • Components, hardware, and configurations.
  • Cost, ROI, and pricing.
  • Standards, security, and interoperability.

WLAN Basics

  • Uses radio waves as carrier.
  • Final link to user is wireless.
  • Backbone generally wired.

How WLANs Work

  • Uses same protocols as wired LAN.
  • Supports same applications.

How WLANs Differ

  • Different physical & MAC layer.
  • Uses Access Points for bridging.
  • Supports roaming.
  • Special security needs.
  • Different hardware & performance.

Common WLAN Topologies

1. WLAN + Wired LAN
  • Access Point (AP) bridges wireless and wired traffic.
  • AP may act as a repeater to extend range.
2. Complete Wireless Network (Ad Hoc)
  • No centralized control.
  • Self-organizing devices.
  • Suitable for temporary events (meetings/conferences).

Physical Layer

  • Wireless NIC scrambles data and modulates radio signals.

Data Link Layer

  • Uses CSMA/CA instead of CSMA/CD.
  • Avoids collisions using channel sensing & RTS/CTS.

CSMA Techniques

Carrier Sense
  • Node checks if channel is free before transmitting.
  • Reduces collisions but cannot eliminate them.
CSMA/CD (Wired)
  • Detect collision → stop → send jam → resend.
  • Used in Ethernet (IEEE 802.3).
CSMA/CA (Wireless)
  • Avoids collision before sending.
  • Uses RTS/CTS handshake.
  • Used in WiFi (IEEE 802.11).

Detailed Comparison: CSMA/CA vs CSMA/CD

Feature CSMA/CA (Wireless) CSMA/CD (Wired)
When Effective Prevents collision before it happens. Handles collision after it occurs.
Used In Wireless networks Wired Ethernet networks
Collision Handling Minimizes the chance of collision Reduces recovery time after collision
Transmission Behavior Sends RTS/CTS (intent to send) before data Sends data → detects conflict → resends
Standard IEEE 802.11 (Wi-Fi) IEEE 802.3 (Ethernet)
Purpose Avoid collision on a shared wireless medium Detect collision on a wired shared medium

6. Hidden Terminal Problem

Cause: Nodes A and C cannot hear each other but both send to B → collision at B.

Solution: RTS/CTS (MACA)
  • A sends RTS to B.
  • B replies with CTS.
  • C hears CTS → waits.
  • A sends data safely.
