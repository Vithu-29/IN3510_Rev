2. Duplexing Techniques

Duplexing: The ability to talk and listen simultaneously. Essential for two-way communication.

Feature FDD (Frequency Division Duplexing) TDD (Time Division Duplexing)
Concept Uses two separate frequency bands: forward (BS → Mobile) and reverse (Mobile → BS). Uses same frequency but different time slots for forward and reverse links.
Hardware Requires a duplexer; transceiver must operate on two frequency bands. No duplexer needed; a simple switch is sufficient.
Full Duplex Supports true full-duplex communication. Not true full-duplex (uplink/downlink alternate in time).
Interference / Coordination Requires strict frequency coordination with out-of-band users. Less complex frequency planning (uses only one band).
Latency Very low latency due to simultaneous TX/RX. Time latency present due to switching between time slots.

3. Narrowband vs. Wideband

Coherence Bandwidth

The range of frequencies over which the channel is considered "flat" (passes components with equal gain).

It represents the maximum frequency interval where two signal components experience similar or correlated fading.

  • Narrowband System: Signal BW < Coherence BW.
    Result: Non-selective (flat) fading – whole signal affected together.
    Tech: Narrowband FDMA (dedicated channel), Narrowband TDMA (shared via time slots).
  • Wideband System: Signal BW > Coherence BW.
    Result: Frequency-selective fading – only parts of the signal fade.
    Tech: CDMA (simultaneous access), Wideband TDMA (time-slot allocation).

4. FDMA, TDMA, CDMA Visualized

Logical Separations

  • FDMA: User assigned a unique frequency channel. No sharing of that frequency.
  • TDMA: Users share same frequency but use unique time slots.
  • CDMA: Users share frequency AND time. Separation via unique Codes.

Combinations

Systems often combine access and duplexing methods:

  • FDMA/FDD
  • FDMA/TDD
  • TDMA/FDD
  • TDMA/TDD

5. CDMA Detailed

Concept: Orthogonal Codes

CDMA allows multiple users to transmit simultaneously on the same frequency by assigning each user a unique orthogonal Walsh code (chipping sequence).

  • Encoded Signal = (Data Bit) × (User’s Walsh Code)
  • All encoded signals are added → transmitted as one summed signal
  • Receiver recovers data by multiplying with its own Walsh code (inner product)
  • Orthogonality ensures no interference between users

Encode: Data × Code

Decode: (Received Sum × User Code)

Example (From Slides)

User A Code: (-1, -1, -1, -1)

User B Code: (-1, +1, -1, +1)

Case: A = 1, B = 1

A signal: (-1, -1, -1, -1)

B signal: (-1, +1, -1, +1)

Sum: (-2, 0, -2, 0)

A decodes: (-2×-1) + (0×-1) + (-2×-1) + (0×-1) = 4 → Bit 1

Practice Question (Solved)

User A Code: (-1, 1, -1, 1) → Bit sequence: 1 0 0
User B Code: (-1,-1, 1, 1) → Bit sequence: 1 1 0
User C Code: (-1, 1, 1,-1) → Bit sequence: 1 0 1

a) Transmitted Summed Sequence (for first bits):

A: (-1, 1, -1, 1)
B: (-1,-1, 1, 1)
C: (-1, 1, 1,-1)
---------------------
Sum: (-3, 1, 1, 1)

b) Decoding (Inner Product):

  • Rx A: (-3,1,1,1) × (-1,1,-1,1) = 3 + 1 - 1 + 1 = 4 → Bit 1
  • Rx B: (-3,1,1,1) × (-1,-1,1,1) = 3 - 1 + 1 + 1 = 4 → Bit 1
  • Rx C: (-3,1,1,1) × (-1,1,1,-1) = 3 + 1 + 1 - 1 = 4 → Bit 1
